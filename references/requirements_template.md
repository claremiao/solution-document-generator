# 信息化解决方案需求信息采集模板

## 使用说明

本模板用于指导用户结构化地提供信息化项目的需求信息。可通过以下方式使用：

1. **交互式采集**：运行 `collect_requirements.cjs` 脚本进行问答式采集
2. **直接填写**：用户按模板结构直接提供文字描述
3. **混合模式**：先提供基础信息，再通过追问补充细节

## 需求信息采集清单

### 1. 项目背景

**采集要点：**
- 项目所属行业及行业现状
- 项目建设政策依据或业务驱动因素（如《中国制造 2025》《"十五五"规划》等）
- 当前面临的痛点或挑战
- 项目建设的必要性

**示例回答：**
```
行业：制造业 - 汽车零部件
现状：行业竞争激烈，数字化转型加速
政策依据：《中国制造 2025》、《"十五五"规划》、《工业互联网创新发展行动计划》
痛点：生产数据孤岛严重、质量追溯困难、设备运维效率低
必要性：提升生产效率、降低运营成本、满足客户质量追溯要求
```

### 2. 目标用户

**采集要点：**
- 主要用户群体及角色
- 用户规模预估
- 用户使用场景
- 特殊用户群体需求

**示例回答：**
```
用户角色：生产操作工、班组长、车间主任、质量管理员、设备管理员、管理层
用户规模：约 500 人
使用场景：车间现场操作、办公室数据分析、移动端巡检
特殊需求：管理层需要移动审批和实时报表
```

### 3. 核心业务需求

**采集要点：**
- 主要业务流程
- 关键业务功能
- 业务数据处理需求
- 与其他系统的业务协同

**示例回答：**
```
业务流程：生产计划->工单下达->生产执行->质量检验->入库管理
关键功能：生产调度、工艺管理、质量管理、设备管理、物料管理
数据需求：实时采集生产数据、质量数据、设备状态数据
系统集成：与 ERP 集成获取物料信息、与 MES 集成传递生产数据
```

### 4. 技术要求

**采集要点：**
- 系统架构偏好（如微服务、单体等）
- 技术栈要求或限制
- 性能指标（并发量、响应时间等）
- 安全性要求
- 可扩展性要求

**示例回答：**
```
架构：微服务架构，支持容器化部署
技术栈：Spring Cloud、Vue.js、MySQL、Redis
性能：并发用户 200+，页面响应<3 秒，支持 7x24 运行
安全：等保二级、数据加密、权限控制
扩展：支持水平扩展、模块化扩展
```

### 5. 预算范围

**采集要点：**
- 项目总投资预算
- 软件采购预算
- 硬件及基础设施预算
- 运维预算

**示例回答：**
```
总投资：500 万元
软件：300 万元（含定制开发）
硬件：150 万元（服务器、网络设备等）
运维：50 万元/年
```

### 6. 实施周期

**采集要点：**
- 项目期望启动时间
- 关键时间节点要求
- 项目总周期
- 分期建设计划（如有）

**示例回答：**
```
启动时间：2025 年 1 月
关键节点：3 月完成需求分析、6 月完成开发、9 月上线试运行
总周期：12 个月
分期：一期核心功能、二期扩展功能
```

### 7. 特殊约束条件

**采集要点：**
- 合规性要求（如等保、行业规范）
- 技术限制条件
- 资源约束
- 其他特殊要求

**示例回答：**
```
合规：等保二级、GMP 规范
技术限制：需兼容现有 Oracle 数据库
资源约束：现有服务器资源有限，需利旧
其他：需支持国产化适配
```

## 信息采集技巧

### 追问策略

1. **模糊信息澄清**：当用户提供"高性能"、"易用"等模糊描述时，追问具体指标
2. **优先级确认**：当需求较多时，引导用户区分优先级
3. **约束条件挖掘**：主动询问可能的技术、资源、时间约束

### 信息验证

- **一致性检查**：验证预算与功能范围是否匹配
- **可行性评估**：评估技术要求与实施周期的合理性
- **完整性确认**：确保关键需求无遗漏

## 输出格式

采集完成后，应输出结构化的 JSON 格式数据：

```json
{
  "projectBackground": {
    "industry": "制造业 - 汽车零部件",
    "drivers": "政策驱动、市场竞争",
    "painPoints": "数据孤岛、质量追溯困难",
    "necessity": "数字化转型需要"
  },
  "targetUsers": {
    "roles": ["操作工", "班组长", "管理层"],
    "scale": 500,
    "scenarios": "车间现场、办公室、移动端"
  },
  ...
}
```

## 常见问题处理

### 用户无法明确需求

**应对策略：**
- 提供行业典型案例参考
- 使用场景化描述引导
- 分阶段确认，先核心后扩展

### 需求频繁变更

**应对策略：**
- 建立需求变更管理流程
- 明确变更影响评估
- 设置需求冻结时间点

### 预算与需求不匹配

**应对策略：**
- 提供分级方案（基础版、标准版、高级版）
- 明确优先级，支持分期建设
- 提供云服务等灵活部署方案
